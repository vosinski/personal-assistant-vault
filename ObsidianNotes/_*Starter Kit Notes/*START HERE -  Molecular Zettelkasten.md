## Syncing Between Mobile & Desktop

I use iCloud to sync between mobile & desktop, which is much easier now with the latest OS & iOS updates where you can keep cloud folders downloaded permanently. 

You can also use other third party sync options, like Google Drive, OneDrive, SyncThing. However, some of these services DO NOT work with iOS or Android or both, so I suggest caution. 

You can also use Obsidian Sync, which is $4 a month for perfect storage syncing and I've only heard good things about it. 

==IMPORTANT== Your mobile vault selection MUST be the root vault folder in order to enable sync. This means that you should add this Molecular Zettelkasten vault to your sync provider, and then choose it as your root folder in the mobile Obsidian App to ensure there are not sync issues. 

## Personalizing Your System
The goal of this system is not to copy exactly what I am doing, but to use my system as a starting point for your own system. Feel free to *change* *anything*! This Molecular Zettelkasten system is meant to be a starting point for your own personal knowledge management. 

One key point is that if you choose to not use the names of "atom", "molecule", "alloy" and would like to rename the notes to your own custom names, some parts of this system ==will break== (e.g., the inboxes listed in the [[Inbox Template]]).

But, since this is a minimalist starter vault, it's very easy for you to update the inboxes to match your changed names. 

All you have to do is change, e.g., #alloy to #publishednote in the [[Alloy (Publishable Note) Inbox]] Dataview query:
![[Alloy Dataview Inbox Query Example.png]]

You can then repeat this for #atom #molecule etc. 

You may also want to change the corresponding folder names, which will change how you set up the [[Auto Note Mover Plugin]] rules below. Note that if you would like to ==move== folders into another folder, such as creating a "Molecular Zettelkasten" folder that contains the "Atoms", "Molecules", "Alloys", etc., you will also need to modify the Auto Note Mover Plugin to update the file path.

That said, I will be continuing to reference these note types as part of the molecular zettelkasten system, so you may find it easier to keep the naming convention the same. Totally up to you!

## Additional Obsidian Settings To Set Up

If you selected "trust author & install plugins" from the beginning, I believe these settings should be turned on for you already. If not, here's a list of additional Obsidian Settings I recommend changing. 

### Daily Notes
Date format: YYYY-MM-DD [Week] W [Day] d shows the Week # and Day # in my daily notes

You can also select the Daily Notes Folder to store new Daily Notes, and the [[Daily Note Template (Basic)]] for the template. 

I also included my own personal [[Daily Note Template (Wanderloots Full)]] as the default daily note to get you started. This note might have too much information for you, so feel free to customize it as you need! My hope was to show you what is possible with Daily Notes & Templater so that you can have all the tools available to you 😊 

### Files & Links
I created an _attachments_ folder to contain all PNGs, JPGS, and other non-markdown files. You can set the file storage location to be this folder under Obsidian Settings.

### Hot Keys
Some of my favourite hot keys I use/set myself. Note that cmd = ctrl for Windows users.
1. Insert Template = option + shift + T 
2. cmd + O = open
3. cmd + N = new note

## Plugins

I want to warn you to be careful about installing too many plugins at the start. It's a rabbit-hole that can very easily lead to many hours spent trying to "perfect" your system rather than *actually using it*. 

My suggestion is to start simple, with as few plugins as possible, and add plugins only as you find a need for them. 

That said, there here are a few plugins that I think are essential and some that I recommend. 

If you clicked "trust author of vault and enable plugins" many of these plugins will already be installed and activated for you 🫡 If you wanted to install the plugins yourself, here is a guide:
